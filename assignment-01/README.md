## 💼 Business Case

### Client

**_TechNova_**

An Artificial Intelligence startup that regularly publishes technical articles for developers, entrepreneurs, and technology enthusiasts.

---

### Business Problem

The client needs a blog article page that can be clearly understood by both search engines and assistive technologies **without relying on CSS**.

The HTML document must communicate the hierarchy of information through semantic elements only, allowing crawlers like Googlebot to accurately identify the article structure.

---

### Project Objective

Build a semantic HTML5 blog page that demonstrates:

- Clean semantic architecture
- Production-ready document structure
- SEO-friendly metadata
- Accessible content hierarchy
- Maintainable HTML foundation
- Future-ready frontend structure

---

## 🧱 Semantic Structure

The page is organized using modern HTML5 semantic elements:

- `<header>`
- `<nav>`
- `<main>`
- `<article>`
- `<section>`
- `<figure>`
- `<picture>`
- `<time>`
- `<footer>`

The article itself contains:

- Blog title
- Publication date
- Featured image
- Multiple content sections
- Author information

## 📝 Reflection

This project reinforces the importance of semantic HTML as the foundation of modern frontend development.

Rather than focusing on visual presentation, the project demonstrates how a well-structured document improves accessibility, search engine understanding, maintainability, and long-term scalability.

By relying exclusively on HTML5 semantic elements, the page remains meaningful even without CSS, making it easier for both users and search engines to interpret the content.
